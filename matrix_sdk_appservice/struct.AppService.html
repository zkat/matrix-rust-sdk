pub async fn virtual_user_client(
    &self,
    localpart: impl AsRef<str>
) -> Result<Client>

Create a Client for the given VirtualUser‚Äôs localpart

Will create and return a Client that‚Äôs configured to assert the identity on all outgoing homeserver requests if localpart is given.

This method is a singleton that saves the client internally for re-use based on the localpart. The cached Client can be retrieved either by calling this method again or by calling Self::get_cached_client() which is non-async convenience wrapper.

Note that if you want to do actions like joining rooms with a virtual user it needs to be registered first. Self::register_virtual_user() can be used for that purpose.

Arguments

  • localpart - The localpart of the user we want assert our identity to

pub async fn set_event_handler(

    handler: Box<dyn EventHandler>
) -> Result<()>

Convenience wrapper around Client::set_event_handler() that attaches the event handler to the MainUser‚Äôs Client

Note that the event handler in the AppService context only triggers join room timeline events, so no state events or events from the invite, knock or leave scope. The rationale behind that is that incoming AppService transactions from the homeserver are not necessarily bound to a specific user but can cover a multitude of namespaces, and as such the AppService basically only ‚Äúobserves joined rooms‚Äù. Also currently homeservers only push PDUs to appservices, no EDUs. There‚Äôs the open MSC2409 regarding supporting EDUs in the future, though it seems to be planned to put EDUs into a different JSON key than events to stay backwards compatible.
